# Plan Template

Use this template to structure the plan output. Write in the user's voice — direct, no fluff. Keep it readable and practical.

---

# <Feature Name>

## Problem

What problem are we solving and why. Written from the user's perspective.

## Solution

How we're solving it. High-level approach, not implementation details.

## Expected Behavior

What the user should be able to do when this is done. Concrete, higher-level descriptions of user-facing behavior.

- User clicks X, Y happens
- When Z is configured, W is available
- etc.

## Implementation Decisions

Key architectural and technical decisions made during the refine session.

- Architecture and system design choices
- Interfaces between components
- Data flow and state management
- API contracts or schema changes
- Patterns and idioms to follow

Do not include specific file paths or code snippets. They go stale fast.

## Testing Approach

How this feature should be tested. What makes a good test for this feature, which parts need tests, and any relevant prior art in the codebase.

## Out of Scope

What we are explicitly not doing as part of this feature.
